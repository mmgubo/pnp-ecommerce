# PnP Ecommerce Service

Self-contained Spring Boot 3 backend providing a product catalogue, order
management, and sales reporting API. Built for the Pick n Pay mid-level Java
take-home assessment.

## Stack

| Concern | Choice |
|---|---|
| Language | Java 17 |
| Framework | Spring Boot 3.2 / Spring MVC |
| Database | H2 in-memory (no external service required) |
| ORM | Spring Data JPA / Hibernate |
| Explicit SQL | NamedParameterJdbcTemplate (reporting queries) |
| Validation | Jakarta Bean Validation |
| Security | Spring Security — HTTP Basic on write endpoints |
| API Docs | Springdoc OpenAPI / Swagger UI |
| Testing | JUnit 5, Mockito, MockMvc, @DataJpaTest, @SpringBootTest |
| Build | Maven 3.8+ |
| Containerisation | Multi-stage Docker + docker-compose |
| CI | GitHub Actions — `mvn -B verify` on push / PR |

## Run

### Maven (local JDK 17)

```
mvn spring-boot:run
```

### Docker Compose

```
docker compose up --build
```

### Tests

```
mvn test
```

## Endpoints

Base path: `http://localhost:8080/api/v1`

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST`  | `/products` | Basic Auth | Create a product |
| `GET`   | `/products?name=&minPrice=&maxPrice=&page=&size=&sort=` | Public | Paginated, filtered product list |
| `GET`   | `/products/{id}` | Public | Single product |
| `POST`  | `/orders` | Basic Auth | Place an order (locks stock, decrements atomically) |
| `GET`   | `/orders/{id}` | Public | Order with items |
| `PATCH` | `/orders/{id}/cancel` | Basic Auth | Cancel a CONFIRMED order, restore stock |
| `GET`   | `/reports/top-products?limit=10&from=&to=` | Public | Top N products by units sold (CONFIRMED only) |
| `GET`   | `/reports/order-summary` | Public | Counts / revenue / avg by status |

## Default credentials

`admin` / `secret`

## Architecture notes

- **Prices in cents.** R39.99 is stored as `3999` (INT). Eliminates IEEE 754
  rounding; the wire response formats as `R%.2f` only at the boundary.
- **Pessimistic write lock on stock.** `findByIdWithLock` issues
  `SELECT ... FOR UPDATE` so concurrent orders cannot both observe sufficient
  stock and race to oversell.
- **Price snapshot on order item.** `unit_price` captures the product price at
  placement time — later catalogue price changes cannot mutate historical orders.
- **Explicit `schema.sql` with `ddl-auto=none`.** All tables, named constraints,
  and indexes are declared by hand instead of generated by Hibernate.
- **JPA + JdbcTemplate split.** CRUD via JPA repositories; reporting
  aggregations via `NamedParameterJdbcTemplate` so the SQL stays explicit.
